# CISSP Frameworks Quick-Lookup

## NIST Publications

NIST SP 800-30: Guide for Conducting Risk Assessments — D1 (Security and Risk Management)
NIST SP 800-37: Risk Management Framework (RMF) — D1 (Security and Risk Management)
NIST SP 800-53: Security and Privacy Controls catalog — D1, D3, D4, D5, D6, D7, D8 (cross-cutting)
NIST SP 800-53A: Assessing Security and Privacy Controls — D6 (Security Assessment and Testing)
NIST SP 800-61: Computer Security Incident Handling Guide — D7 (Security Operations)
NIST SP 800-63: Digital Identity Guidelines — D5 (Identity and Access Management)
NIST SP 800-88: Media Sanitization Guidelines — D2 (Asset Security), D7 (Security Operations)
NIST SP 800-115: Technical Guide to Information Security Testing — D6 (Security Assessment and Testing)
NIST SP 800-137: Information Security Continuous Monitoring — D6, D7 (Assessment, Operations)
NIST CSF 2.0: Cybersecurity Framework (Govern, Identify, Protect, Detect, Respond, Recover) — D1, D3, D7 (cross-cutting)
NIST SSDF: Secure Software Development Framework — D8 (Software Development Security)

## ISO/IEC Standards

ISO/IEC 27001: Information Security Management Systems (ISMS) requirements — D1 (governance), D6 (audit)
ISO/IEC 27002: Code of practice for information security controls — D1, D3 (controls guidance)
ISO/IEC 27005: Information security risk management — D1 (Security and Risk Management)
ISO/IEC 27033: Network security — D4 (Communication and Network Security)
ISO/IEC 27034: Application security — D8 (Software Development Security)
ISO/IEC 27035: Incident management — D7 (Security Operations)
ISO/IEC 15408: Common Criteria for IT security evaluation — D3 (Security Architecture and Engineering)

## Industry Frameworks

COBIT: IT governance framework — D1 (Security and Risk Management)
ITIL: IT service management framework — D7 (Security Operations)
OWASP Top 10: Top web application security risks — D8 (Software Development Security)
OWASP SAMM: Software Assurance Maturity Model — D8 (Software Development Security)
OWASP Testing Guide: Web application security testing methodology — D6, D8 (Assessment, Development)

## Regulations and Standards

GDPR: EU General Data Protection Regulation — D1 (legal/compliance), D2 (data rights/retention)
HIPAA: US Health Insurance Portability and Accountability Act — D1 (compliance), D2 (PHI protection)
SOX: Sarbanes-Oxley Act (financial reporting controls) — D1 (compliance), D6 (audit)
PCI DSS: Payment Card Industry Data Security Standard — D2 (cardholder data), D4 (network segmentation), D6 (ASV scans)
FIPS 140-2/3: Cryptographic module validation standard — D3 (Security Architecture and Engineering)
FERPA: Family Educational Rights and Privacy Act — D1 (compliance), D2 (student data)
GLBA: Gramm-Leach-Bliley Act (financial privacy) — D1 (compliance), D2 (customer data)
